    Observability¶

    HyperBinder includes built-in observability that you can enable with a single line. When tracing is on, every operation automatically emits events and records timing data. No manual wiring of event buses, subscribers, or trace contexts required.

    hb.tracing = True
hb.unbind([1, 2], ["subject"], collection="knowledge_graph")

for event in hb.event_history:
    print(event.event_type.value, event.data.get("duration_ms"))

    Enabling Tracing¶

    Tracing is off by default and has zero overhead when disabled. Enable it on any client instance:

    from hybi import HyperBinder

hb = HyperBinder(url="http://localhost:8000", api_key="...")

# Property setter
hb.tracing = True

# Or use methods
hb.enable_tracing()
hb.disable_tracing()

    Each client has its own tracing state. Enabling tracing on one client does not affect any other.

    hb1 = HyperBinder(url="http://localhost:8000", api_key="...")
hb2 = HyperBinder(url="http://localhost:8000", api_key="...")

hb1.tracing = True
print(hb2.tracing)  # False

    Event History¶

    When tracing is enabled, the client records up to 1,000 recent events. Each event includes a type, timestamp, and operation-specific data.

    hb.tracing = True
hb.unbind([1, 2], ["subject"], collection="kg")

for event in hb.event_history:
    dur = event.data.get("duration_ms")
    extra = f"  ({dur:.1f} ms)" if dur else ""
    print(f"{event.event_type.value}{extra}")

    Output:

    unbind.started
unbind.completed  (12.3 ms)

    Every instrumented operation emits a pair of events:

    Operation Started Event Completed Event Failed Event
    unbind unbind.started unbind.completed unbind.failed
    extract extract.started extract.completed extract.failed
    traverse_fuzzy traverse.started traverse.completed traverse.failed
    analogy analogy.started analogy.completed analogy.failed

    Completed events include duration_ms and result_count. Failed events include error and error_type.

    Scoped Traces¶

    Use hb.trace() to group a sequence of operations into a single trace with nested spans and timing. Tracing is automatically enabled for the duration of the block and restored to its previous state afterwards.

    with hb.trace("find_collaborators") as t:
    results = hb.unbind([1], ["subject"], collection="kg")
    neighbors = hb.traverse_fuzzy(
        "Einstein", "subject", ["student_of"], collection="kg",
    )
    analogies = hb.analogy("Einstein", "Relativity", "Darwin", collection="kg")

# Inspect the span tree
for span in t.get_spans():
    dur = f"{span.duration_ms:.1f} ms" if span.duration_ms else ""
    print(f"  {span.name}  {dur}")

    Output:

      find_collaborators  45.2 ms
    unbind  8.1 ms
    traverse_fuzzy  22.4 ms
    analogy  14.3 ms

    The trace is also stored on the client for later retrieval:

    print(hb.last_trace.trace_id)
print(hb.last_trace.to_dict())  # full serializable representation

    Nested Traces¶

    Traces can be nested. Each hb.trace() block creates an independent trace context:

    with hb.trace("outer") as outer:
    hb.unbind([1], ["subject"], collection="kg")
    with hb.trace("inner") as inner:
        hb.unbind([2], ["object"], collection="kg")

# Each trace has its own span tree
print(len(outer.get_spans()))  # includes outer root + unbind + inner root + unbind
print(len(inner.get_spans()))  # includes inner root + unbind

    Event Callbacks¶

    Register a callback to react to events in real time. This is how you connect HyperBinder tracing to external monitoring systems.

    hb.tracing = True

# Register a callback  returns an unsubscribe function
unsub = hb.on_event(lambda e: print(f"[{e.event_type.value}]"))

hb.unbind([1], ["subject"], collection="kg")
# prints: [unbind.started]
# prints: [unbind.completed]

# Stop receiving events
unsub()

    Monitoring Failures¶

    failures = []

def on_failure(event):
    if "failed" in event.event_type.value:
        failures.append(event)

hb.on_event(on_failure)

    Logging Integration¶

    import logging

log = logging.getLogger("hyperbinder.trace")

def log_events(event):
    dur = event.data.get("duration_ms")
    if dur and dur > 100:
        log.warning("Slow operation: %s (%.0f ms)", event.event_type.value, dur)
    elif "failed" in event.event_type.value:
        log.error("Operation failed: %s  %s", event.event_type.value, event.data.get("error"))

hb.on_event(log_events)

    Schema Visualization¶

    The graph export utilities convert molecule schemas into visual representations. These are standalone functions that work independently of client tracing.

    from hybi.compose import Triple, Pair, Field, Encoding
from hybi.observe.graph import molecule_to_graph, to_mermaid

schema = Triple(
    subject=Pair(
        left=Field("entity_type", encoding=Encoding.EXACT),
        right=Field("entity_name"),
    ),
    predicate=Field("relation", encoding=Encoding.EXACT),
    object=Field("target"),
)

graph = molecule_to_graph(schema)
print(to_mermaid(graph))

    The output is a valid Mermaid diagram that you can paste into mermaid.live or render directly in mkdocs:

    flowchart TB
    root["Triple"]
    pair_subject["Pair"]
    field_entity_type(("entity_type (EXACT)"))
    field_entity_name(("entity_name (SEMANTIC)"))
    field_relation(("relation (EXACT)"))
    field_target(("target (SEMANTIC)"))
    root -->|subject| pair_subject
    pair_subject -->|left| field_entity_type
    pair_subject -->|right| field_entity_name
    root -->|predicate| field_relation
    root -->|object| field_target

    Export Formats¶

    Function Output Use Case
    to_mermaid() Mermaid flowchart Documentation, markdown rendering
    to_dot() GraphViz DOT High-quality PDF/SVG via dot CLI
    to_json_graph() JSON Graph Format (JGF) Programmatic consumption, web UIs

    Intersection Registry Graphs¶

    You can also visualize collection relationships from an intersection registry:

    from hybi.compose.intersections import IntersectionRegistry, Intersection
from hybi.observe.graph import registry_to_graph, to_mermaid

registry = IntersectionRegistry()
registry.add(Intersection("users.id", "orders.user_id"))
registry.add(Intersection("orders.product_id", "products.id"))

graph = registry_to_graph(registry)
print(to_mermaid(graph))

    API Reference¶

    Client Properties and Methods¶

    Member Type Description
    hb.tracing bool Get or set whether tracing is active
    hb.enable_tracing() method Enable event emission
    hb.disable_tracing() method Disable event emission
    hb.event_history list Most recent events (up to 1,000)
    hb.on_event(fn) method Register a callback; returns unsubscribe function
    hb.trace(name) context mgr Scoped trace with nested spans
    hb.last_trace TraceContext or None Most recent completed trace

    Event Object¶

    Field Type Description
    event_type EventType The type of event (enum)
    timestamp float Unix timestamp
    trace_id str or None Trace correlation ID
    data dict Operation-specific metadata

    TraceContext Object¶

    Member Type Description
    trace_id str Unique trace identifier
    get_spans() method Returns list of TraceSpan objects
    to_dict() method Serializable dict representation

    Graph Export Functions¶

    Function Input Output
    molecule_to_graph() Molecule schema Graph dict
    traversal_to_graph() TraversalTrace Graph dict
    registry_to_graph() IntersectionRegistry Graph dict
    to_dot(graph) Graph dict DOT string
    to_mermaid(graph) Graph dict Mermaid string
    to_json_graph(graph) Graph dict JGF JSON string
